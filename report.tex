\begin{abstract}
This report describes the development of an evaluation framework to assess the capability of LLMs (especially Qwen) to perform literature analysis and recognization, combined with reasoning, across multiple languages. For testing the data, from a well known and well trained source literature was taken from Project Gutenberg. The program evaluates the model's capacity to process text snippets and accurately identify the corresponding book from a set of random possible titles. 
Analyzed were english, german, french, dutch and italian literature and cross-lingual test between german and english, where german texts were matched to english titles, checking the models capability of applying translated texts. The question datasets were automatically created by using text processing for the header each book in Project Gutenberg has. 
Additionaly the model's reasoning capabilities were tested, were we manually analyzed the models reasons for choosing certain titles. The checking of the model results were also done automatically, via regex. 
At last we corrupted the text data, by introducing noise into them, and testing the models robustness. All of our evaluations are visualized in graphics, showing our results. 
\end{abstract}

\subsection{Database}
For the database, we searched through Hugging Face for appropriate literature databases to use. First we wanted to use nikolina-p/gutenberg\_clean\_en\_splits\footnote{https://huggingface.co/datasets/nikolina-p/gutenberg\_clean\_en\_splits}, because the dataset had only books in english already and it had stripped all headers of the books. At first this seemed like a good approach to make sure the model would only be asked about actual text parts, and not accidentally some part of the header which did not contribute to the book itself. Unfortunatly the books in this database were only identifiable by the numbers anymore as title and similar things were contained in the header. That is why we switched to manu/project\_gutenberg\footnote{https://huggingface.co/datasets/manu/project_gutenberg/viewer/default/en}, which was the original database used by nikolina-p/gutenberg\_clean\_en\_splits.
